# 前端未落地项梳理与实施计划

基于 `docs/project-audit.md`（2026-03-15）的审计结果，以下列出“前端未落地”的功能点，并给出实施计划与优先级。该计划仅覆盖前端层面未落地或占位的内容。

## 一、未落地项清单（来自审计）

### P1（功能缺口/体验影响明显）
1. **前台订单物流与售后入口（占位）**
   - 位置：`frontend/src/views/front/OrderList.vue`、`frontend/src/views/front/OrderDetail.vue`
   - 现状：展示按钮/区域为占位，无真实数据或交互。

2. **后台退款/物流面板（占位）**
   - 位置：`frontend/src/views/admin/OrderManage.vue`
   - 现状：面板为占位，未接入后端逻辑。

3. **后台用户角色面板（占位）**
   - 位置：`frontend/src/views/admin/UserManage.vue`
   - 现状：角色相关操作/面板未落地。

4. **后台优惠券/公告（占位）**
   - 位置：`frontend/src/views/admin/BannerManage.vue`
   - 现状：面板为占位，无对应接口。

5. **后台仪表盘统计、设置、日志（静态占位）**
   - 位置：`frontend/src/views/admin/Dashboard.vue`、`Settings.vue`、`Logs.vue`
   - 现状：静态展示，无数据驱动。

### P2（接口/页面覆盖完善）
6. **后台用户详情接口未落地前端调用**
   - 接口：`GET /api/admin/users/{id}`
   - 现状：前端 API 模块存在，但无页面调用。
   - 影响：后台用户仅有列表与状态切换，缺少详情入口。

### P3（配置一致性，前端弱相关）
7. **CORS 旧端口配置混杂（非前端实现）**
   - 位置：`application.properties`
   - 说明：后端配置层问题，非前端功能实现项。

---

## 二、实施计划（建议优先级）

### 1) 快速补齐前端可落地项（P2 优先）
**目标：** 用最小改动完成“已有接口但前端未调用”的落地。
- 增加后台用户详情入口（例如列表“查看详情”按钮）
- 复用 `frontend/src/api/admin/user.ts` 的详情接口
- 新增或扩展详情弹窗/抽屉组件

**建议文件：**
- `frontend/src/views/admin/UserManage.vue`
- `frontend/src/api/admin/user.ts`

---

### 2) 评估占位功能是否需要真实落地（P1）
**目标：** 明确是否需要把占位功能变为真实功能。
- 前台订单物流/售后入口：是否有后端接口/数据？
- 后台订单退款/物流面板：是否计划支持退款/物流？
- 后台用户角色面板：是否需要角色管理功能？
- 优惠券/公告：是否需要新模块支持？
- 仪表盘/设置/日志：是否需要统计与系统配置？

**执行方式：**
- 若后端无接口 → 先确认需求，再评估是否补后端
- 若仅 UI 占位 → 可直接移除或改为“即将上线”提示

---

### 3) 结合产品目标补充数据驱动页（P1/P2）
**目标：** 将“静态页/占位页”替换为真实数据。
- Admin Dashboard：新增统计 API 后接入
- Settings/Logs：根据后端可用信息决定是否落地或移除

---

## 三、推荐落地顺序（建议）
1. **后台用户详情页面/弹窗**（已有接口，成本最低）
2. **前台订单物流/售后入口**（需明确后端数据来源）
3. **后台订单退款/物流面板**（需明确业务流程）
4. **仪表盘统计/设置/日志**（需新增后端支持）
5. **优惠券/公告**（需新增模块）
