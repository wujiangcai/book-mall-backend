## 附录：完整的订单状态流转图

```
┌─────────────┐
│  待支付(0)   │ ← 订单创建
└──────┬──────┘
       │
       ├─────→ 用户支付 ─────┐
       │                    │
       │                    ↓
       │              ┌─────────────┐
       │              │  已支付(1)   │ ← 支付成功（瞬时状态）
       │              └──────┬──────┘
       │                     │
       │                     │ 自动变更（同一事务）
       │                     ↓
       │              ┌─────────────┐
       │              │  待发货(2)   │ ← 等待管理员发货
       │              └──────┬──────┘
       │                     │
       │                     │ 管理员发货
       │                     ↓
       │              ┌─────────────┐
       │              │  已发货(3)   │ ← 等待用户收货
       │              └──────┬──────┘
       │                     │
       │                     │ 用户确认收货或自动完成
       │                     ↓
       │              ┌─────────────┐
       │              │  已完成(4)   │ ← 订单完成
       │              └─────────────┘
       │
       └─────→ 用户取消或超时 ─────┐
                                  │
                                  ↓
                           ┌─────────────┐
                           │  已取消(5)   │ ← 订单取消
                           └─────────────┘
```

**状态变更权限：**
- 待支付 → 已支付：用户支付操作
- 已支付 → 待发货：系统自动（支付成功后立即执行）
- 待发货 → 已发货：管理员发货操作
- 已发货 → 已完成：用户确认收货或系统自动（发货后7-15天）
- 待支付 → 已取消：用户主动取消或系统自动（创建后24小时未支付）
